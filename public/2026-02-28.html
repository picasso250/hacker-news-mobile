2. Windows 95 用户界面：可用性工程案例研究 (1996)

得票数: 34

原文链接: View Article

总结： 这是一篇经典的可用性研究报告，回顾了 Windows 95 界面设计背后的严谨测试。与现代流行的“扁平化”设计不同，Windows 95 的设计初衷是让用户通过视觉暗示（如阴影和凸起）清晰地识别可点击元素。文章引发了人们对现代 UI 设计趋势（如 macOS Tahoe 过度圆角化）的集体反思，认为现代设计往往为了美学而牺牲了实用性和清晰度。

热门评论翻译：

设计师往往比开发者更难接受反馈。这解释了为什么尽管扁平化 UI 被证明存在可用性缺陷却依然流行。每一年的可用性都在下降，比如 macOS Tahoe 明显不如 Sequoia。别指望苹果会修复那些过度圆角的窗口。
看看那是多么清晰、专业且好用。这篇报告写得很好。在那样的测试和投入下，绝对不会搞出今天 Windows 这种糟糕的局面。Mac OS 也在加速倒退，他们刚从 Jony Ive 的风格中恢复一点，又让包装设计师来管 UI，搞出了“流体玻璃”那种烂摊子。
这就是典型的“如果没坏就别修它”。Windows 95 奠定了现代桌面交互的基础。
